# 開發日誌

## 2026-05-30

### 今日目標

- 根據實際試用回饋，改善「互動式專案發想教練」的發想品質。
- 讓系統從單純產生三個點子，逐步變成能幫使用者收斂方向、避開普通題目、提供下一步規劃的 project coach。

### 已完成項目

#### Phase 1: 強化問卷收斂能力

- 新增 `docs/優化計畫.md`，將後續優化拆成多個可實作階段。
- 將問卷從 3 步擴充為 4 步，新增「收斂條件」：
  - 不想做的類型或技術
  - 主要目標使用者
  - 可用資料來源
  - 成功標準
  - 偏好成果形式
  - 課程要求或必須展示的概念
- 將新增欄位串接到 `/api/generate-ideas` input schema。
- 將新增欄位加入 prompt，讓 LLM 能根據限制條件生成更貼近使用者需求的專案點子。
- 手動測試 Phase 1 問卷流程，確認新增欄位可正常操作。

#### Phase 2: 升級 Prompt 與 Output Schema

- 強化 prompt 規則：
  - 每個提案必須引用使用者輸入條件。
  - 三個提案必須在使用情境、技術亮點、資料結構或演算法上有差異。
  - 避免把登入 / 註冊當作預設 MVP。
  - 若題目容易變成通用題，必須說明差異化設計。
- 擴充 LLM output schema，新增：
  - `why_this_is_not_generic`
  - `core_algorithm_or_data_structure`
  - `first_3_steps`
  - `week_plan`
  - `scope_down_plan`
  - `demo_highlight`
- 更新結果卡片，顯示差異化原因、核心資料結構 / 演算法、前三個實作步驟、分週開發計畫、縮 scope 方案與 Demo 亮點。
- 更新 schema 測試資料，確保 LLM 回傳格式缺少必要欄位時會被測試抓出。

  - 測試輸入可生成包含核心資料結構 / 演算法、前三步與 Demo 亮點的專案提案
